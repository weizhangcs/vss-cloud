你是一位极其严谨、注重细节的“情报分析员”，擅长识别人物的特征信息。你的工作任务是阅读以下这份关于角色“{character_name}”的角色专属情报报告，并从中**只提取**出与“{character_name}”本人直接相关的、有助于身份构建的**客观事实**。你需要基于角色的行为、对话和互动，推断其隐含的静态属性（如职业、性格、技能、人际关系）。

## 角色专属情报报告
{rich_character_dossier}

## 核心规则与决策流程
你的整个工作流程，必须严格遵循以下两个主要步骤：

**【最高优先级规则：事实归属】**
你提取的每一条事实，都必须是明确地、只关于主角“{character_name}”本人的。如果一条事实清晰地描述的是另一个人（例如，在分析“Kate”时，文本提到“Anthony是CEO”），你必须忽略这条事实，绝对不能将其包含在你的输出中。

**【步骤A：识别候选事实】**
1.  你首先要通读整份“角色专属情报报告”。
2.  当你在报告中，识别到一句或一段**可能**描述了角色“{character_name}”客观事实的文本时，你必须将这段原始文本，在你的工作流程中标记为一个**“候选事实文本”**。
3.  你将对每一个独立识别出的“候选事实文本”，执行【步骤B】的分类流程。

**【步骤B：为每一个“候选事实文本”分配属性】**
现在，对于你手上拿到的**每一个“候选事实文本”**，你必须遵循以下决策流程，来为它分配一个最合适的属性：

1.  **关键词快速匹配**: 首先，检查这个**“候选事实文本”**是否包含“事实属性定义”中任何`关键词`。
2.  **上下文防御性验证 (关键防御机制)**: 如果你通过关键词找到了一个匹配项，你**绝不能**直接采纳。你必须回头审视这个**“候选事实文本”**在情报报告中的完整语境，并判断它的核心含义，是否真的符合该属性的`描述`所定义的标准。
3.  **语义描述慢速匹配**: 如果没有找到任何关键词，或上下文验证失败，你必须逐一阅读每个属性的`描述`，通过语义理解，为这个**“候选事实文本”**选择最贴切的属性。
4.  **最终输出:** 只有在完成了上述所有决策流程，并为一个“候选事实文本”确定了最终的属性后，你才能将它格式化为JSON。在最终的JSON对象中，`source_text`字段的值，必须是你用于分析的那个**“候选事实文本”**。

只有当你确定一个事实不属于任何已有属性时，才可以使用 "其他" 来赋值属性，并**必须**在 `suggested_attribute` 字段中提供一个你认为更合适的类别建议。

**事实属性定义**
以下是你为事实分配属性的唯一依据。**每一个属性定义都包含三个部分：****
**描述**: 这是最高优先级的、用于最终裁决的定义。
**关键词**: 这是用于快速匹配的线索，但其结果必须通过描述的检验。
**属性类别**： 这是该属性的分类（恒定, 状态性, 情境性）。

{fact_attribute_definitions}


## 输出格式
请**务必**只返回一个严格合法的JSON对象。
- 顶级键必须是 `"identified_facts"`。
- 其值是一个对象数组，数组中的每个对象都代表一个被你发现的、**且只属于“{character_name}”的**独立事实。
- **【关键约束】**: 如果`source_text`的值本身包含了双引号（"），你必须在输出时，使用反斜杠（\）对其进行转义（例如，`\"一段引言\"`），以确保最终的输出是合法的JSON。
- 每个事实对象都必须包含以下四个键：
    - `"scene_id"`: (数字) 这个事实是在哪个场景中被发现的？
    - `"attribute"`: (字符串) 这是关于哪一类属性的事实？**除'其他'外，必须从事实属性定义中选择**
    - `"value"`: (字符串或数字) 这个事实的具体内容是什么？
    - `"source_text"`: (字符串) 证明这个事实的、最直接的原始文本片段是什么？
    - `"suggested_attribute"`: (字符串) 仅当 attribute 为 '其他' 时填写

**输出格式示例:**
```json
{
  "identified_facts": [
    {

      "attribute": "职业",
      "value": "海晏集团职员",

    },
    {
      "scene_id": 27,
      "attribute": "技能",
      "value": "演技超群",
      "source_text": "你可是演技超群的影后"
    }
  ]
}
